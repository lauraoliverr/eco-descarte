# Arquitetura do Sistema - Eco Descarte

## 1. Tecnologias Utilizadas

- Frontend: TypeScript, HTML, CSS
- Backend: (não utilizado ou futuro uso com Node.js)
- Banco de Dados: (não utilizado inicialmente)
- Hospedagem: Web (GitHub Pages ou similar)

## 2. Modelo Arquitetural

O sistema foi estruturado com base no modelo C4, utilizando uma arquitetura simples orientada a aplicações web.

No nível de containers, o sistema é composto por:

- Frontend: responsável pela interface com o usuário, onde são exibidas informações sobre descarte correto de resíduos.
- (Futuro) Backend: poderá ser implementado para fornecer dados dinâmicos e integração com APIs externas.

O sistema funciona de forma direta, onde o usuário acessa a aplicação web e interage com as informações disponíveis.

## 3. Justificativa da Arquitetura

A escolha de uma arquitetura simples foi feita devido ao caráter inicial e educativo do projeto.

Essa abordagem permite:

- Facilidade de desenvolvimento
- Baixa complexidade
- Facilidade de manutenção
- Possibilidade de expansão futura (com backend e banco de dados)

Além disso, o uso de tecnologias web torna o sistema acessível para qualquer usuário com acesso à internet.
